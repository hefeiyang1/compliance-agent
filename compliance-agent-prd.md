# 企业内部智能合规问答与 Agent 助手系统 - PRD

## 项目概述

**项目名称**：ComplianceAgent - 企业内部智能合规问答与 Agent 助手系统
**项目类型**：Python + LangGraph + RAG + LLM 微调
**开发周期**：2024.05 - 2024.10
**目标**：构建基于大语言模型的企业内网合规问答系统，实现自然语言提问、法规检索与引用型回答

---

## 核心功能需求

### 1. RAG 检索增强模块
- 企业合规文档向量化存储（支持 PDF、Word、Markdown）
- 语义相似度检索（使用 Embedding 模型）
- 检索结果重排序与上下文拼装
- 引用来源标注与追溯

### 2. Agent 智能体架构（P-E-R）
**Plan（规划阶段）**
- 用户意图识别与任务拆解
- 多步骤任务规划与依赖分析
- 检索策略选择（向量检索/关键词检索/混合）

**Execute（执行阶段）**
- 工具调用与知识检索
- 中间结果缓存与状态管理
- 异常处理与重试机制

**Review（审查阶段）**
- 结果质量评估（完整性、准确性、合规性）
- 引用验证与来源标注
- 自我反思与迭代优化

### 3. MCP 工具网关
- 统一工具调用接口（标准化输入输出）
- 工具注册与发现机制
- 参数校验与异常拦截
- 调用日志与监控

### 4. 模型 API 调用层
- 统一的模型调用接口（支持 OpenAI 兼容格式）
- 多模型支持（企业内网模型/公有云 API）
- 智能路由与负载均衡
- Prompt 模板管理
- 调用失败重试与降级策略

## 技术难点与解决方案

### 难点 1：长文档任务的上下文丢失
**问题**：在处理多步骤任务时，模型容易丢失早期信息，导致意图漂移

**解决方案**：
- 实现 LangGraph State 分区管理（short_term / long_term / context）
- 动态上下文拼装策略（根据任务类型选择相关上下文）
- 关键信息持久化与动态加载

### 难点 2：多工具调用的参数混乱
**问题**：模型幻觉导致工具调用参数格式错误，调用成功率低

**解决方案**：
- 引入 MCP 协议，定义标准工具调用格式
- 参数校验层（JSON Schema 验证）
- 异常拦截与重试机制

### 难点 3：模型在合规场景的表现不足
**问题**：通用模型在严谨语气、指令遵循、逻辑推理上表现不佳

**解决方案**：
- **短期**：优化 Prompt 设计，使用思维链（CoT）引导模型
- **中期**：针对合规场景构建高质量的 Prompt 模板库
- **长期（可选）**：积累数据后，考虑模型微调（需要 GPU 资源和专业知识）

**Prompt 优化策略**：
- 角色设定：明确"合规专家"身份
- 思维链引导："让我们一步步分析..."
- 约束条件：强调"基于检索到的法规回答"
- 引用要求："请标注信息来源"
